# 🚗 렌트카 대여 관리 프로그램 (Rent-A-Car Management System)

> **2024년 데이터베이스 캡스톤디자인 결과보고서**
> 데이터베이스(DB) 설계와 구축을 중점으로, 렌터카 업체 운영자가 차량 및 회원을 효율적으로 관리하고 사용자가 편리하게 차량을 대여·반납할 수 있도록 개발한 웹 기반 관리 프로그램입니다.

* **효율적인 자원 관리**: 렌터카 사업자가 복잡한 차량 상태(대여 가능 여부, 정비 상태 등)와 고객 대여 이력, 비용 정산 등을 한눈에 파악하고 제어할 수 있는 직관적인 관리 환경을 제공합니다.

## 3. 📐 데이터베이스 설계 (Database Design)

이 프로젝트는 데이터의 일관성과 확장성을 보장하기 위해 체계적인 3단계 DB 설계 과정을 거쳤습니다.

### 3-1. 개념적 설계 (Conceptual Design)
* 핵심 엔티티(Entity) 분석을 통해 회원(User), 차량(Car), 대여(Renting), 결제/비용(Cost), 관리자(Admin) 간의 관계를 정의하고 개체-관계 모델(ERD)을 도출했습니다.

### 3-2. 논리적 설계 (Logical Design) 및 정규화 (Normalization)
* 삽입/삭제/수정 이상 현상(Anomaly)을 방지하기 위해 릴레이션 스키마를 정비하고, **제3정규형(3NF)**까지 정규화를 프로세스를 단계별로 적용하여 테이블을 분해 및 최적화했습니다.

### 3-3. 주요 테이블 구조
* `user` (회원 테이블): 회원 고유 ID, 비밀번호, 이름, 연락처 등 관리
* `car` (차량 테이블): 차량 번호(Car_id), 모델명, 대여 가격, 차량 이름 및 대여 가능 상태 관리
* `renting` (대여 테이블): 대여 신청한 회원 ID, 차량 ID, 대여 시작일(Date_st), 대여 종료일(Date_end) 관리
* `cost` (비용/이력 테이블): 반납이 완료된 대여 건의 최종 결제 금액 및 이력 누적 관리

## ✨ 4. 주요 기능 및 프로세스 검증

### 👤 1. 회원 및 차량 대여 프로세스
* **대여 신청**: 사용자가 차량을 선택해 대여를 신청하면 `renting` 테이블에 유저 ID와 차량 ID가 `INSERT`되며, 대여 시작일과 종료일은 관리자 승인 전까지 `NULL` 상태로 대기합니다.
* **관리자 승인 및 대여 시작**: 관리자가 대여 요청을 확인하고 승인하면, `Date_st`에 현재 날짜가 등록되며 차량 상태가 '대여 중'으로 전환됩니다.

### 🔄 2. 반납 및 비용 정산 프로세스
* **반납 처리**: 차량 반납 버튼을 누르면 해당 대여 정보가 `renting` 테이블에서 안전하게 `DELETE`됩니다.
* **이력 이관**: 삭제와 동시에 데이터의 보존을 위해 트랜잭션 처리를 거쳐 `cost` 테이블로 대여 기간 및 비용 정보가 자동으로 `INSERT`되어 정산 이력으로 관리됩니다.

### 🛠️ 3. 관리자 전용 기능 (차량 무결성 제어)
* **차량 추가**: 새 차량의 번호(Car_id), 모델, 가격을 입력하면 `car` 테이블에 중복 여부를 PK(기본키)로 검증한 후 `INSERT`됩니다.
* **차량 삭제**: 관리자가 특정 차량 번호를 입력하여 리스트에서 안전하게 제거할 수 있습니다.
